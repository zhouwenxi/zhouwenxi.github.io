Android ButterKnife 【黄油刀】学习使用

发表于 2018-04-10 | 分类于 Android | 阅读次数: |

简介

ButterKnife 用于绑定事件，绑定资源，简化代码，提升开发效率。

使用

A:配套插件

首先安装Android ButterKnife Zelezny插件，如果在鼠标移动到布局文件上，
点右键没有出现Generate ButterKnife Injections,那么就可能是没有依赖库ButterKnife。

B:依赖

implementation 'com.jakewharton:butterknife:8.8.1'
annotationProcessor 'com.jakewharton:butterknife-compiler:8.8.1'

i:在Activity中使用

public class MainActivity extends AppCompatActivity {

    @BindView(R.id.main_tv0)
    TextView mainTv0;
    @BindView(R.id.main_tv1)
    TextView mainTv1;
    @BindView(R.id.main_tv2)
    TextView mainTv2;
    @BindView(R.id.main_tv3)
    TextView mainTv3;
    @BindView(R.id.main_btn0)
    Button mainBtn0;
    @BindView(R.id.main_btn1)
    Button mainBtn1;
    @BindView(R.id.main_btn2)
    Button mainBtn2;
    @BindView(R.id.main_ll)
    LinearLayout mainLl;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        ButterKnife.bind(this);
        initEvent();
    }

    private void initEvent() {

        mainTv0.setText("100");
        mainTv1.setText("101");
        mainTv2.setText("102");
        mainTv3.setText("103");

    }

    @OnClick({R.id.main_btn0, R.id.main_btn1, R.id.main_btn2})
    void click(View view) {

        switch (view.getId()) {
            case R.id.main_btn0:
                Toast.makeText(this, "main_btn0", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn1:
                Toast.makeText(this, "main_btn1", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn2:
                Toast.makeText(this, "main_btn2", Toast.LENGTH_SHORT).show();
                break;
            default:
                break;
        }

    }

    @OnLongClick(R.id.main_ll)
    boolean clickLinear() {
        Toast.makeText(this, "main_ll", Toast.LENGTH_SHORT).show();
        return true;
    }
}

在布局文件上点击Generate ButterKnife Injections就可以生成相关代码。

注意 ButterKnife.bind(this);在加载布局之后，控件不能设置private或者static

ii:在fragment中使用

public class MainFragment extends Fragment {

    @BindView(R.id.main_tv0)
    TextView mainTv0;
    @BindView(R.id.main_tv1)
    TextView mainTv1;
    @BindView(R.id.main_tv2)
    TextView mainTv2;
    @BindView(R.id.main_tv3)
    TextView mainTv3;
    @BindView(R.id.main_btn0)
    Button mainBtn0;
    @BindView(R.id.main_btn1)
    Button mainBtn1;
    @BindView(R.id.main_btn2)
    Button mainBtn2;
    Unbinder unbinder;

    @Nullable
    @Override
    public View onCreateView(LayoutInflater inflater, @Nullable ViewGroup container, @Nullable Bundle savedInstanceState) {

        View view = inflater.inflate(R.layout.fragment_main, null);

        unbinder = ButterKnife.bind(this, view);
        return view;
    }

    @Override
    public void onActivityCreated(@Nullable Bundle savedInstanceState) {
        super.onActivityCreated(savedInstanceState);

        initEvent();
    }

    private void initEvent() {

        mainTv0.setText("fg100");
        mainTv1.setText("fg101");
        mainTv2.setText("fg102");
        mainTv3.setText("fg103");


    }


    @OnClick({R.id.main_btn0, R.id.main_btn1, R.id.main_btn2})
    void click(View view) {

        switch (view.getId()) {
            case R.id.main_btn0:
                Toast.makeText(getActivity(), "fg_main_btn0", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn1:
                Toast.makeText(getActivity(), "fg_main_btn1", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn2:
                Toast.makeText(getActivity(), "fg_main_btn2 去绑定资源", Toast.LENGTH_SHORT).show();

                startActivity(new Intent(getActivity(), SecondActivity.class));

                break;
            default:
                break;
        }

    }

    @Override
    public void onDestroyView() {
        super.onDestroyView();
        unbinder.unbind();
    }
}

使用插件可以一键生成，但是需要注意的是

unbinder = ButterKnife.bind(this, view);绑定布局，其中this不能替换为getActivity()
unbinder.unbind();在销毁的时候解除绑定。

iii:绑定view

public class ViewHolder {

    @BindView(R.id.main_tv0)
    TextView mainTv0;
    @BindView(R.id.main_tv1)
    TextView mainTv1;
    @BindView(R.id.main_tv2)
    TextView mainTv2;
    @BindView(R.id.main_tv3)
    TextView mainTv3;
    @BindView(R.id.main_btn0)
    Button mainBtn0;
    @BindView(R.id.main_btn1)
    Button mainBtn1;
    @BindView(R.id.main_btn2)
    Button mainBtn2;

    private View view;
    private Activity mActivity;

    public ViewHolder(Activity activity) {
        this.mActivity = activity;
        view = View.inflate(activity, R.layout.fragment_main, null);
        ButterKnife.bind(this, view);
    }

    @OnClick({R.id.main_btn0, R.id.main_btn1, R.id.main_btn2})
    public void onViewClicked(View view) {
        switch (view.getId()) {
            case R.id.main_btn0:
                Toast.makeText(mActivity, "main_btn0", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn1:
                Toast.makeText(mActivity, "main_btn1", Toast.LENGTH_SHORT).show();
                break;
            case R.id.main_btn2:
                Toast.makeText(mActivity, "main_btn2", Toast.LENGTH_SHORT).show();
                break;
            default:
                break;
        }
    }

    public View getView() {
        return view;
    }
}

其实与绑定fragment类似，都是绑定其中的view ；即:ButterKnife.bind(this, view);

iv:绑定资源

public class SecondActivity extends AppCompatActivity {

    //绑定单个
    @BindView(R.id.second_tv0)
    TextView secondTv0;
    @BindView(R.id.second_btn0)
    Button secondBtn0;
    @BindView(R.id.second_iv)
    ImageView iv;

    //一次绑定多个
    @BindViews({R.id.second_tv1, R.id.second_tv2, R.id.second_tv3})
    List<TextView> mListTv;

    //绑定资源
    @BindString(R.string.app_name)
    String appName;
    @BindArray(R.array.test)
    String[] str;
    @BindBitmap(R.mipmap.ic_launcher)
    Bitmap bitmap;
    @BindColor(R.color.colorAccent)
    int color;


    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_second);
        ButterKnife.bind(this);

        initEvent();
    }

    private void initEvent() {

        secondTv0.setText(appName);

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < str.length; i++) {
            sb.append(str[i]).append(" ");
        }
        mListTv.get(0).setText(sb.toString());

        mListTv.get(1).setTextColor(color);

    }

    @OnClick({R.id.second_tv0, R.id.second_tv1, R.id.second_tv2, R.id.second_tv3, R.id.second_btn0})
    public void onViewClicked(View view) {
        switch (view.getId()) {
            case R.id.second_tv0:
                break;
            case R.id.second_tv1:
                break;
            case R.id.second_tv2:
                break;
            case R.id.second_tv3:
                iv.setImageBitmap(bitmap);
                break;
            case R.id.second_btn0:
                startActivity(new Intent(SecondActivity.this, MainView.class));
                break;
            default:
                break;
        }
    }
}

上面代码所示，butterknife不仅仅是在处理findViewById和onclick上，依旧可以绑定其他资源，如String、String[]、Bitmap、Drawable、color等；还可以依次绑定多个同种类型的控件，多个点击事件。还有许多用法没有写上，如CheckBox改变监听，item点击监听等，但是如果不清楚的话可以使用findviewbyid再设置相应事件，并不冲突!

2、处理ViewHolder代码

public class MainView extends AppCompatActivity {

    @BindView(R.id.mv_fl)
    FrameLayout mvFl;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main_view);
        ButterKnife.bind(this);
        initEvent();
    }

    private void initEvent() {
        View view = new ViewHolder(this).getView();
        mvFl.addView(view);
    }
}

最后

尽管ButterKnife很方便使用，但是毋庸置疑还是有许多人使用findviewbyid方式。
其中也有相关的插件FindViewByMe或者是相关将布局文件转换成id形式如

http://android.lineten.net/layout.php

写布局再转换也很方便，但是点击事件，绑定资源却无能为力，不是那么便捷。
可能在性能上使用bufferknife不如原始的findviewbyid
但对于一个项目来说可以忽略不计。
重要的是，如果接手别人的项目时，不至于茫然失措，可以做到心中有数。
